2. 通过创建 p5.js 生成艺术表达（.html + .js 文件）

首先，执行此任务：

## 算法哲学创建

首先，创建一个算法哲学（而非静态图像或模板），它将通过以下方式被诠释：
- 计算过程、涌现行为、数学之美
- 种子随机性、噪声场、有机系统
- 粒子、流场、力场
- 参数变化和受控混沌

### 核心理解
- 接收的内容：用户提供的一些微妙输入或指令，作为基础考虑，但不应限制创作自由。
- 创建的内容：一个算法哲学/生成美学运动。
- 接下来发生的事：同一版本接收该哲学并对其进行代码表达——创建 90% 算法生成、10% 必要参数的 p5.js 草图。

考虑这种方式：
- 为一个生成艺术运动撰写宣言
- 下一阶段是编写将该运动变为现实的算法

哲学必须强调：算法表达。涌现行为。计算之美。种子变化。

### 如何生成算法哲学

**命名该运动**（1-2 个词）："有机湍流" / "量子谐波" / "涌现静默"

**阐述哲学**（4-6 段 — 简洁但完整）：

为捕捉算法本质，表达该哲学如何通过以下方式体现：
- 计算过程和数学关系？
- 噪声函数和随机模式？
- 粒子行为和场动态？
- 时间演化和系统状态？
- 参数变化和涌现复杂性？

**关键指南：**
- **避免冗余**：每个算法方面只提及一次。避免重复关于噪声理论、粒子动力学或数学原理的概念，除非增加新深度。
- **反复强调工艺**：哲学必须多次强调最终算法应该看起来像是花费了无数小时开发的、精心打磨的、出自计算美学领域绝对顶尖人物之手。这种框架至关重要——重复使用诸如"精心制作的算法"、"深厚计算专业知识的结晶"、"煞费苦心的优化"、"大师级实现"等表述。
- **留出创作空间**：保持算法方向的具体性，但足够简洁，让下一个版本有空间以极高工艺水平做出诠释性实现选择。

哲学必须引导下一个版本以算法方式表达想法，而非通过静态图像。美存在于过程中，而非最终帧。

### 哲学示例

**"有机湍流"**
哲学：受自然法则约束的混沌，从无序中涌现的秩序。
算法表达：由分层 Perlin 噪声驱动的流场。数千粒子沿向量力流动，其轨迹积累成有机密度图。多个噪声八度创建湍流区域和宁静区域。颜色从速度和密度中涌现——快速粒子燃烧明亮，缓慢的褪入阴影。算法运行直至平衡——精心调校的平衡，每个参数都经过计算美学大师无数次迭代精炼。

**"量子谐波"**
哲学：离散实体展现类波干涉模式。
算法表达：粒子在网格上初始化，每个携带通过正弦波演化的相位值。当粒子靠近时，它们的相位干涉——建设性干涉创建明亮节点，破坏性创建虚空。简谐运动生成复杂的涌现曼陀罗。煞费苦心的频率校准之成果，每个比率都经过精心选择以产生共振之美。

**"递归低语"**
哲学：跨尺度的自相似性，有限空间中的无限深度。
算法表达：递归细分的分支结构。每个分支略带随机但受黄金比例约束。L 系统或递归细分生成既数学又有机的树状形态。微妙的噪声扰动打破完美对称。线宽随每个递归级别递减。每个分支角度都是深度数学探索的结果。

**"场动力学"**
哲学：通过力对物质的影响使不可见的力可见。
算法表达：由数学函数或噪声构造的向量场。粒子在边缘出生，沿场线流动，到达平衡或边界时死亡。多个场可以吸引、排斥或旋转粒子。可视化仅显示痕迹——不可见力的幽灵般证据。一场通过力平衡精心编排的计算之舞。

**"随机结晶"**
哲学：随机过程结晶为有序结构。
算法表达：随机化圆形填充或 Voronoi 镶嵌。从随机点开始，让它们通过松弛算法演化。单元格推开直至平衡。颜色基于单元格大小、邻居数量或与中心的距离。涌现的有机镶嵌既感觉随机又不可避免。每个种子产生独特的结晶之美——大师级生成算法的标志。

*这些是简化的示例。实际的算法哲学应为 4-6 个充实的段落。*

### 核心原则
- **算法哲学**：创建一个将通过代码表达的计算世界观
- **过程优先于产品**：始终强调美从算法的执行中涌现——每次运行都是独特的
- **参数表达**：想法通过数学关系、力、行为传达——而非静态构图
- **艺术自由**：下一个版本以算法方式诠释哲学——提供创意实现空间
- **纯粹生成艺术**：这是关于创建活的算法，而非带随机性的静态图像
- **专家工艺**：反复强调最终算法必须感觉像是精心制作的、经过无数次迭代精炼的、计算美学领域绝对顶尖人物深厚专业知识的结晶

**算法哲学应为 4-6 段长。** 用富有诗意的计算哲学填充，将预期愿景凝聚在一起。避免重复同一点。将此算法哲学输出为 .md 文件。

**关键步骤**：在实现算法之前，识别原始请求中的微妙概念线索。

**核心原则**：

### 技术要求

**种子随机性（Art Blocks 模式）**：
```javascript
// 始终使用种子以确保可重现性
let seed = 12345; // 或用户输入的哈希值
randomSeed(seed);
noiseSeed(seed);
```

**参数结构 — 遵循哲学**：

为建立从算法哲学中自然涌现的参数，思考："这个系统的哪些品质可以调整？"

```javascript
let params = {
  seed: 12345,  // 始终包含种子以确保可重现性
  // 颜色
  // 添加控制你的算法的参数：
  // - 数量（多少？）
  // - 尺度（多大？多快？）
  // - 概率（多可能？）
  // - 比例（什么比例？）
  // - 角度（什么方向？）
  // - 阈值（行为何时改变？）
};
```

**为设计有效参数，关注系统需要可调的属性，而非以"图案类型"来思考。**

**核心算法 — 表达哲学**：

**关键**：算法哲学应决定构建什么。

为通过代码表达哲学，避免思考"我应该使用哪种图案？"，而应思考"如何通过代码表达这个哲学？"

如果哲学关于**有机涌现**，考虑使用：
- 随时间积累或生长的元素
- 受自然规则约束的随机过程
- 反馈循环和交互

如果哲学关于**数学之美**，考虑使用：
- 几何关系和比率
- 三角函数和谐波
- 精确计算创造意想不到的图案

如果哲学关于**受控混沌**，考虑使用：
- 严格边界内的随机变化
- 分岔和相变
- 从无序中涌现的秩序

**算法从哲学中流出，而非从选项菜单中选择。**

为指导实现，让概念本质通知创意和原创选择。构建表达这个特定请求愿景的东西。

**画布设置**：标准 p5.js 结构：
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 初始化你的系统
}

function draw() {
  // 你的生成算法
  // 可以是静态的（noLoop）或动画的
}
```

### 工艺要求

**关键**：为达到精通水平，创建感觉像是生成艺术大师经过无数次迭代才产出的算法。精心调校每个参数。确保每个图案都有目的地涌现。这不是随机噪声——这是经过深厚专业知识精炼的受控混沌。

- **平衡**：复杂性而非视觉噪声，秩序而非刻板
- **色彩和谐**：深思熟虑的调色板，而非随机 RGB 值
- **构图**：即使在随机性中，也保持视觉层次和流动
- **性能**：流畅执行，动画时优化为实时
- **可重现性**：相同种子始终产生相同输出

### 输出格式

输出：
1. **算法哲学** — 作为 Markdown 或文本解释生成美学
2. **单个 HTML 制品** — 从 `templates/viewer.html` 构建的自包含交互式生成艺术（见步骤 0 和下一节）

## 交互式制品创建

**提醒：`templates/viewer.html` 应该已经被阅读（见步骤 0）。将该文件作为起点。**

为允许探索生成艺术，创建单个自包含的 HTML 制品。确保此制品在 claude.ai 或任何浏览器中立即运行 — 无需设置。将所有内容内联嵌入。

### 关键：什么是固定的 vs 可变的

`templates/viewer.html` 文件是基础。它包含所需的确切结构和样式。

**固定的（始终与原样完全包含）：**
- 布局结构（头部、侧边栏、主画布区域）
- Anthropic 品牌（UI 颜色、字体、渐变）
- 侧边栏中的种子区域：
  - 种子显示
  - 上一个/下一个按钮
  - 随机按钮
  - 跳转到种子输入 + 前往按钮
- 侧边栏中的操作区域：
  - 重新生成按钮
  - 重置按钮

**可变的（为每件作品自定义）：**
- 整个 p5.js 算法（setup/draw/类）
- 参数对象（定义艺术需要什么）
- 侧边栏中的参数区域：
  - 参数控件数量
  - 参数名称
  - 滑块的最小/最大/步进值
  - 控件类型（滑块、输入框等）
- 颜色区域（可选）：
  - 某些艺术需要颜色选择器
  - 某些艺术可能使用固定颜色
  - 某些艺术可能是单色的（不需要颜色控件）
  - 根据艺术需求决定

**每件作品应有独特的参数和算法！** 固定部分提供一致的 UX — 其他一切都表达独特的愿景。

### 必需功能

**1. 参数控件**
- 数值参数的滑块（粒子数、噪声尺度、速度等）
- 调色板的颜色选择器
- 参数更改时实时更新
- 重置按钮恢复默认值

**2. 种子导航**
- 显示当前种子号
- "上一个"和"下一个"按钮循环种子
- "随机"按钮获取随机种子
- 输入框跳转到特定种子
- 请求时生成 100 个变体（种子 1-100）

**3. 单制品结构**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- p5.js 从 CDN - 始终可用 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 所有样式内联 - 干净、简约 */
    /* 画布在上，控件在下 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 所有参数控件 -->
  </div>
  <script>
    // 所有 p5.js 代码内联在此
    // 参数对象、类、函数
    // setup() 和 draw()
    // UI 处理程序
    // 一切自包含
  </script>
</body>
</html>
```

**关键**：这是单个制品。无外部文件，无导入（p5.js CDN 除外）。一切内联。

**4. 实现细节 — 构建侧边栏**

侧边栏结构：

**1. 种子（固定）** — 始终与原样完全包含：
- 种子显示
- 上一个/下一个/随机/跳转按钮

**2. 参数（可变）** — 为艺术创建控件：
```html
<div class="control-group">
    <label>参数名称</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
根据参数数量添加任意多个 control-group div。

**3. 颜色（可选/可变）** — 如果艺术需要可调颜色则包含：
- 如果用户应控制调色板，添加颜色选择器
- 如果艺术使用固定颜色，跳过此区域
- 如果艺术是单色的，跳过

**4. 操作（固定）** — 始终与原样完全包含：
- 重新生成按钮
- 重置按钮
- 下载 PNG 按钮

**要求**：
- 种子控件必须工作（上一个/下一个/随机/跳转/显示）
- 所有参数必须有 UI 控件
- 重新生成、重置、下载按钮必须工作
- 保持 Anthropic 品牌（UI 样式，而非艺术颜色）

### 使用制品

HTML 制品立即可用：
1. **在 claude.ai 中**：作为交互式制品显示 — 立即运行
2. **作为文件**：保存并在任何浏览器中打开 — 无需服务器
3. **分享**：发送 HTML 文件 — 它完全自包含

## 变体与探索

制品默认包含种子导航（上一个/下一个/随机按钮），允许用户在不创建多个文件的情况下探索变体。如果用户想要突出显示特定变体：

- 包含种子预设（"变体 1：种子 42"、"变体 2：种子 127"等按钮）
- 添加"画廊模式"，并排显示多个种子的缩略图
- 全部在同一个制品内

这就像从同一印版创建系列印刷——算法一致，但每个种子揭示其潜力的不同面向。交互性意味着用户通过探索种子空间发现他们自己的最爱。

## 创作过程

**用户请求** → **算法哲学** → **实现**

每个请求都是独特的。过程包括：

1. **解读用户意图** — 追求什么美学？
2. **创建算法哲学**（4-6 段）描述计算方法
3. **用代码实现** — 构建表达该哲学的算法
4. **设计适当参数** — 什么应该可调？
5. **构建匹配的 UI 控件** — 这些参数的滑块/输入框

**常量**：
- Anthropic 品牌（颜色、字体、布局）
- 种子导航（始终存在）
- 自包含 HTML 制品

**其他一切都是可变的**：
- 算法本身
- 参数
- UI 控件
- 视觉结果

为获得最佳结果，信任创造力，让哲学指导实现。
